The Core Idea
Instead of saving to the database before preview, pass the current unsaved theme state directly to the preview page without touching the database at all.

🏗️ Ways to Implement This
1 — SessionStorage (Simplest & Best)
On the settings page (before opening preview tab):
javascriptconst handleLivePreview = () => {
  // Store current unsaved theme in sessionStorage
  sessionStorage.setItem('preview_theme', JSON.stringify(localTheme))
  
  // Open preview tab immediately - no async needed
  window.open(`${siteUrl}?preview=true`, '_blank')
}
On the customer page:
javascript// In your ThemeContext or page component
const isPreview = searchParams.get('preview') === 'true'

if (isPreview) {
  const previewTheme = sessionStorage.getItem('preview_theme')
  if (previewTheme) {
    return JSON.parse(previewTheme) // use this instead of API data
  }
}
Why this works:

No database call needed
No async = no popup blocker issue
SessionStorage is shared across tabs in the same browser session
Zero delay — opens instantly
