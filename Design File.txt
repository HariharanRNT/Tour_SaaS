Implement a Multi-Theme System for the TourSaaS customer-facing pages and agent portal. Do not touch any admin pages.
1. Create /lib/themes.ts
Define 7 theme objects with keys: default, orange, purple, teal, blue, rose, amber. Each object has: name, primary, primaryLight, primarySoft, gradientStart, gradientMid, gradientEnd, glow (primary at 25% opacity), preview (3 color array).
Values:

default (current design — must match exactly what is live now): #F97316 / #FDBA74 / #FED7AA — label it "Default" and mark it as the baseline. This is what the app looks like before any theme is applied. Selecting this always restores the original orange/peach glassmorphism design exactly.
orange: #F97316 / #FDBA74 / #FED7AA
purple: #9333EA / #C084FC / #F3E8FF
teal: #0D9488 / #5EEAD4 / #CCFBF1
blue: #2563EB / #93C5FD / #DBEAFE
rose: #E11D48 / #FB7185 / #FFE4E6
amber: #D97706 / #FCD34D / #FEF3C7

The default theme object values must be identical to the current hardcoded colors in the app so that selecting Default is visually indistinguishable from the original design.

#FDBA74 → var(--primary-light)

4. Create /context/ThemeContext.tsx

useState('default') for activeTheme — default is 'default' not 'orange'
useEffect on activeTheme change → document.documentElement.style.setProperty() for all 7 CSS variables using values from /lib/themes.ts
When activeTheme === 'default', apply the same values as orange — this ensures clicking Default always restores the original design
Two separate localStorage keys:

Customer theme → localStorage key 'customer-theme'
Agent theme → localStorage key 'agent-theme'


On mount restore from correct localStorage key
Export ThemeProvider (accepts storageKey prop) and useTheme hook

5. Wrap customer layout and agent layout separately

Agent root layout → <ThemeProvider storageKey="agent-theme">
Do NOT add ThemeProvider to admin layout
