▶ Gemini API call — intent extraction Prompt engineering

This is the system prompt to send to Gemini 2.5 Flash. It must return a strict JSON object — no markdown, no prose.

SYSTEM PROMPT (send as system role to Gemini 2.5 Flash):

You are a travel package filter extraction engine for a tour booking platform.
Your ONLY job is to parse a user's natural-language travel query and return a single valid JSON object — no markdown, no explanation, no extra text, just the raw JSON.

Extract the following fields. Use null for any field not mentioned or unclear:

{
  "destination": string | null,          // city or country name  e.g. "Japan", "Goa", "Mumbai"
  "country": string | null,              // ISO country name if separable from destination
  "minBudget": number | null,            // minimum budget in INR (₹)
  "maxBudget": number | null,            // maximum budget in INR (₹). If user says "minimum X" treat X as minBudget and set maxBudget to null
  "minDays": number | null,              // minimum trip duration in days
  "maxDays": number | null,              // maximum trip duration in days
  "nights": number | null,               // number of nights if specifically mentioned
  "tripStyle": string[] | null,          // e.g. ["beach", "adventure", "cultural", "luxury", "budget", "family", "honeymoon", "solo"]
  "activities": string[] | null,         // e.g. ["hiking", "snorkeling", "sightseeing", "food tour", "temple visit"]
  "packageType": "single_city" | "multi_city" | null,
  "travelMonth": string | null,          // e.g. "May", "next month" → resolve to actual month name if possible
  "groupSize": number | null             // number of travellers
}

Rules:
- If user says "4 days", set minDays=4 and maxDays=4.
- If user says "4-6 days", set minDays=4 and maxDays=6.
- If user says "minimum ₹17000 budget", set minBudget=17000 and maxBudget=null.
- Always return valid JSON. Never return explanatory prose.
- If the entire query is unrelated to travel, return: {"error": "not_travel_query"}
▶ Frontend API handler — call Gemini & build redirect URL JavaScript / Next.js

Call Gemini using the existing env API key, parse the JSON response, map each field to a URL query param, then push to /plan-trip.

Create a utility function `parseAISearchAndRedirect(userQuery, router)` in `/lib/aiSearch.js`.

STEPS:
1. Read NEXT_PUBLIC_GEMINI_API_KEY (or the existing env var name already used in this project for Gemini) from process.env.

2. Call the Gemini 2.5 Flash API:
   - Model: gemini-2.5-flash (use the same model string already in the project)
   - Send the system prompt (see section 2) as a system instruction.
   - Send the userQuery as the user message.
   - Set temperature: 0.1 for deterministic output.
   - Parse the response text as JSON (strip any accidental backtick fences first).

3. MAP JSON fields → URL query params:
   destination       → ?destination=Japan
   country           → ?country=Japan
   minBudget         → ?minPrice=17000
   maxBudget         → ?maxPrice=30000
   minDays           → ?minDays=4
   maxDays           → ?maxDays=4
   tripStyle[]       → ?tripStyle=beach&tripStyle=adventure
   activities[]      → ?activities=hiking&activities=snorkeling
   packageType       → ?packageType=single_city  OR  multi_city
   travelMonth       → ?month=May
   groupSize         → ?groupSize=2

4. Build the URL: /plan-trip?search=all&{params}
   Always include search=all so the existing page logic triggers a full listing.

5. Call router.push(builtUrl) to navigate.

6. Error handling:
   - If JSON contains "error: not_travel_query" → show inline modal error: "Please describe a travel plan."
   - If the Gemini call fails (network / quota) → show inline modal error with the raw message.
   - Wrap in try/catch, never crash the page.

Export the function as default. Keep API key only server-side if using Next.js API routes; otherwise use the existing pattern in this codebase for Gemini calls.
